# 心理健康助手 - MiniMind 实现指南

本项目基于 MiniMind 超轻量语言模型，构建具备共情能力和危机识别功能的心理健康助手。

---

## 一、项目概述

### 1.1 应用场景

- **情绪支持**：提供情感陪伴和倾听
- **心理科普**：普及心理健康知识
- **压力疏导**：帮助缓解日常压力
- **资源推荐**：推荐专业心理咨询资源

### 1.2 核心特性

| 特性 | 说明 |
|------|------|
| 共情能力 | 能识别并回应用户情绪 |
| 危机识别 | 能识别危机情况并建议转介专业机构 |
| 隐私保护 | 本地化部署，数据不外泄 |
| 轻量级 | 模型仅26M，适合边缘部署 |

### 2.3 训练流程

#### 步骤1：预训练（学习基础知识）

```bash
cd trainer
python train_pretrain.py \
    --epochs 1 \
    --batch_size 2 \
    --max_seq_len 128 \
    --save_interval 5
```

#### 步骤2：SFT微调（学习对话方式）

```bash
python train_full_sft.py \
    --epochs 1 \
    --batch_size 2 \
    --max_seq_len 128 \
    --save_interval 5
```

#### 步骤3：心理健康 LoRA 微调（学习领域知识）

```bash
python train_lora.py \
    --data_path ../dataset/mental_health_sft.jsonl \
    --lora_name mental_health \
    --epochs 3 \
    --batch_size 2 \
    --learning_rate 1e-4 \
    --max_seq_len 128 \
    --save_interval 5
```

### 2.4 测试模型

```bash
# 返回项目根目录
cd ..

# 测试心理健康助手
python eval_llm.py --weight full_sft --lora_weight mental_health
```

## 五、模型效果优化

### 5.1 提升效果的方法

| 方法 | 说明 | 预期提升 |
|------|------|----------|
| 增加数据量 | 使用完整数据集（20,000+条） | +30-50% |
| 延长训练时间 | 增加 epochs 到 5-10 | +10-20% |
| 数据增强 | 同义句替换、改写 | +10-15% |
| 混合训练 | 混合多个领域数据 | +5-10% |

### 5.2 训练参数调优

    --lora_name mental_health \
    --epochs 5 \

    --max_seq_len 256 \

## 八、常见问题

### Q1：模型回答质量不高怎么办？

A：这是正常的，示例数据集只有13条数据。建议使用完整数据集（20,000+条）重新训练。

### Q2：训练需要多长时间？

A：
- 使用示例数据：约5-10分钟
- 使用完整数据：约2-5小时（单卡3090）

### Q3：需要什么硬件配置？

A：
- 最低配置：单卡RTX 3090（24GB显存）
- 推荐配置：多卡RTX 3090/4090

### Q4：如何评估模型效果？

A：
1. 人工评估：测试典型对话场景
2. 自动化评估：计算困惑度（Perplexity）
3. 用户测试：邀请同学试用并反馈
